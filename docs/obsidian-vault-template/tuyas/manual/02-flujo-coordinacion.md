# 02 — Flujo del coordinador (día a día)

## Lo que hace el coordinador

Tu rol como coordinador de PPS es **el puente** entre tres mundos: la facultad,
las instituciones externas y los estudiantes. Operativamente:

1. Recibís solicitudes de alumnos que quieren hacer PPS.
2. Gestionás el vínculo con instituciones (existentes y nuevas).
3. Armás lanzamientos cuando una institución confirma cupos.
4. Hacés seguimiento durante la PPS activa.
5. Cerrás la acreditación cuando finaliza.

Y todo eso, en paralelo, sobre ~50-60 instituciones activas.

## Las tres fuentes por las que entra trabajo

### A) Panel web (`https://...`)

- **Solicitudes nuevas** de alumnos.
- **Acciones admin** (cambiar estados, ver historial).
- **Lanzador**: armar una nueva convocatoria.
- Fuente principal de **datos estructurados**.

### B) Gmail (`blas.rivera@uflouniversidad.edu.ar`)

- Hilos con instituciones (formal: convenios, propuestas, derivaciones).
- Hilos con autoridades UFLO (Florencia, directoras, vicerrectora).
- Hilos con alumnos (consultas, cambios, documentación).
- Notificaciones de plataforma (ignorables).

### C) WhatsApp (lista **PPS** en tu celu)

- Conversaciones rápidas con instituciones (confirmaciones, ajustes, dudas).
- Conversaciones internas con autoridades UFLO (Agostina directora,
  Fabiana vicerrectora, etc.).
- Instituciones nuevas todavía sin convenio (charlas iniciales).

## Cadencia típica

| Momento             | Qué hacés                                                             |
| ------------------- | --------------------------------------------------------------------- |
| Inicio cuatrimestre | Re-lanzamientos masivos, contactar nuevas, revisar convenios vencidos |
| Mitad cuatrimestre  | Inscripciones abiertas, selección, generación de seguros              |
| Cierre cuatrimestre | Acreditaciones, cierre de cohortes, notas de seguimiento              |
| Todo el tiempo      | Responder mails/WhatsApp, agendar reuniones, resolver imprevistos     |

## Tus puntos de dolor (lo que justifica a Hermes)

- Demasiados canales en paralelo (panel + Gmail + WhatsApp).
- Cosas que se caen entre las grietas (solicitudes viejas sin responder).
- Cambios de contexto frecuentes — costoso volver a recordar dónde estaba cada institución.
- Mucho conocimiento implícito que no está escrito (justamente por eso este manual).
